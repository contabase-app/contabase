# ContaBase v0.1.0-beta.1 — Beta pública controlada

> **Status:** Publicada
>
> **Estágio:** Beta pública controlada

## Público-alvo

Esta Beta é destinada a pessoas que operam infraestrutura self-hosted e aceitam validar backups, restore, proxy e atualizações antes de usar dados importantes.

Ela pode ser avaliada em uso próprio, familiar ou em pequeno negócio, mas não oferece promessa de estabilidade de produção nem substitui controles contábeis, fiscais ou jurídicos.

## Principais recursos

- contas, lançamentos, cartões e faturas;
- metas, limites e reservas;
- workspaces separados;
- dashboard e relatórios financeiros;
- autenticação, 2FA e recuperação administrativa local;
- backup e restore operacionais;
- imagem Docker pública multi-arch via GHCR;
- artifacts Linux `amd64` e `arm64` com checksum obrigatório;
- build local/source para desenvolvimento e customização.

## Segurança e privacidade

O ContaBase é self-hosted, não possui telemetria e não envia dados financeiros para terceiros.
Os dados, chaves, backups, proxy, domínio e HTTPS ficam sob responsabilidade do operador.

Esta Beta inclui controles de autenticação, sessão, CSRF, isolamento por workspace, headers de segurança e proteção do setup inicial. Isso não representa garantia absoluta nem auditoria externa completa.

## Instalação

### Docker via GHCR

Canal público recomendado para teste rápido e ambiente isolado:

```yaml
services:
  contabase:
    image: ghcr.io/contabase-app/contabase:v0.1.0-beta.1
```

- `v0.1.0-beta.1`: tag fixa recomendada;
- `beta`: canal mutável;
- `latest`: não publicado na Beta.

A imagem está publicada no GHCR como `ghcr.io/contabase-app/contabase:v0.1.0-beta.1` e `ghcr.io/contabase-app/contabase:beta`.

### Release artifact

Recomendado para VPS/LXC Debian sem Docker, em Linux `amd64` ou `arm64`.
Não exige Go ou Node.

A GitHub Release contém os assets listados abaixo:

```bash
curl -fsSLo /tmp/contabase-install.sh \
  https://raw.githubusercontent.com/contabase-app/contabase/v0.1.0-beta.1/scripts/install.sh
less /tmp/contabase-install.sh
chmod +x /tmp/contabase-install.sh
sudo env CONTABASE_INSTALL_METHOD=release CONTABASE_VERSION=v0.1.0-beta.1 \
  /tmp/contabase-install.sh
```

O bootstrap baixa o helper exclusivamente da mesma tag pública. O instalador valida o SHA-256, a estrutura do tarball e a versão do bundle antes de instalar.

### Source/build local

Modo avançado para desenvolvimento e customização:

```bash
git clone --branch v0.1.0-beta.1 https://github.com/contabase-app/contabase.git
cd contabase
chmod +x scripts/install-contabase-source.sh
sudo ./scripts/install-contabase-source.sh
```

Exige Go na versão do `go.mod` e Node.js 20+.

## Checksums

O arquivo `checksums.txt` deve conter SHA-256 para os dois tarballs.
O instalador de release rejeita asset ausente, checksum divergente, conteúdo inesperado, links ou paths inseguros.

## Limitações conhecidas

- **B4 — recorrências e parcelamentos:** a auditoria crítica completa desse fluxo permanece pendente;
- **B5 — pagamento parcial de faturas:** a lógica continua em validação e o fluxo não deve ser tratado como estável;
- atualização automática por release artifact ainda não está implementada;
- não há promessa de compatibilidade estável entre Betas;
- esta versão não é indicada como sistema contábil ou fiscal.

## Backup obrigatório

Antes de instalar ou atualizar:

1. faça backup do banco, uploads e configuração;
2. preserve as chaves usadas pela instância;
3. teste o restore em ambiente separado;
4. valide o healthcheck após a operação.

## Atualizações futuras

As próximas versões terão notas próprias e instruções compatíveis com o método de instalação.
Enquanto não existir um updater público de release artifact, a atualização desse modo deve ser uma reinstalação controlada, com backup, preservação de dados e validação manual.

Consulte [Atualização](../atualizacao.md) antes de trocar de versão.
