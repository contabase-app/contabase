# ContaBase v0.1.0-beta.2 — Beta pública controlada

> **Status:** Em preparação; ainda não publicada
>
> **Estágio:** Beta pública controlada, não stable

## Principais mudanças desde v0.1.0-beta.1

### Adicionado
- **Busca de categorias sem acento:** em Configurações > Categorias, o campo de busca ignora acentos e diacríticos via normalização Unicode (NFD), permitindo encontrar "Saúde" ao digitar `saude`, "Educação" com `educacao` e "Cartão" com `cartao`.
- **Testes de regressão para edição de séries recorrentes:** 13 testes cobrindo alteração de data nos escopos single, future e all.
- **Testes de regressão para limite disponível de cartão:** 14 testes cobrindo cálculo de limite com compras únicas, parcelamentos, recorrências e isolamento por workspace.
- **Estados derivados de fatura por ciclo e situação financeira:** badges independentes como `Aberta`, `Fecha em X dias`, `Fechada`, `Vazia`, `Pendente`, `Parcial` e `Paga`.
- **Seletor manual de fatura no Form Lançamento** para compras no cartão, com opção automática por ciclo e escolha avançada.

### Corrigido
- **Duplicidade ao editar lançamentos recorrentes com mudança de data:** ao alterar a data de um lançamento recorrente nos escopos "somente este", "este e futuros" ou "toda a série", o sistema agora move corretamente as datas das ocorrências, recalcula as faturas vinculadas e preserva a projeção futura sem duplicar registros.
- **Cálculo de limite usado/disponível do cartão:** o limite em uso agora considera todas as despesas comprometidas em faturas abertas (OPEN) ou fechadas (CLOSED) ainda não pagas, incluindo parcelas de compras futuras e projeções de recorrentes materializadas em faturas. Faturas pagas não consomem limite.
- **Cálculo de limite de cartão em faturas com pagamento parcial.**
- **Fatura quitada antecipadamente:** corrigido caso em que era pulada e lançamentos do mesmo ciclo caíam indevidamente na próxima fatura.
- **Travamento/deadlock no endpoint `/cartoes/fatura-destino/...`** ao carregar opções com `include_options=true`.
- **Loop/flicker das ações rápidas de fatura no Form Lançamento.**
- **Badge financeiro `Parcial`:** aparecia como `Pendente` em faturas parcialmente pagas.

### Alterado
- **Redesign visual do financeiro denso (Dense Financial UI):** aplicado padrão visual com cards densos, chips, barras de progresso, linhas compactas e tokens visuais nas principais telas incluindo Dashboard, Metas, Lançamentos, Faturas, Relatórios, Contas, Contatos, Ajuda e Configurações.
- **Scripts de atualização revisados:** scripts privados de deploy passaram a espelhar a branch remota com `git fetch` + `git reset --hard`. Scripts públicos preservam comportamento seguro com fast-forward ou `--force-reset` explícito.
- **Cálculo canônico de limite de cartão:** criada função compartilhada `sumCardOutstandingLimit` que soma despesas em faturas OPEN/CLOSED isoladas por workspace e cartão.
- **Limite usado/disponível do cartão** passou a ser calculado pelo saldo pendente das faturas, e não mais pelo total bruto das compras.
- **Pagamentos parciais** liberam limite proporcionalmente.
- **Quitação total de fatura** zera o consumo de limite daquela fatura.
- **Fatura paga deixou de ser bloqueio rígido** para correções manuais.
- **Roteamento automático de compras no cartão** passou a considerar o ciclo da fatura, não apenas o status `OPEN`.

## Segurança e privacidade

O ContaBase é self-hosted, não possui telemetria e não envia dados financeiros para terceiros.
Os dados, chaves, backups, proxy, domínio e HTTPS ficam sob responsabilidade do operador.

## Instalação

### Docker via GHCR

```yaml
services:
  contabase:
    image: ghcr.io/contabase-app/contabase:v0.1.0-beta.2
```

- `v0.1.0-beta.2`: tag fixa recomendada;
- `beta`: canal mutável;
- `latest`: não publicado na Beta.

### Release artifact

Recomendado para VPS/LXC Debian sem Docker:

```bash
curl -fsSLo /tmp/contabase-install.sh https://get-contabase.pages.dev/install.sh && bash /tmp/contabase-install.sh
```

Ou com versão explícita:

```bash
curl -fsSLo /tmp/contabase-install.sh https://get-contabase.pages.dev/install.sh && CONTABASE_VERSION=vX.Y.Z bash /tmp/contabase-install.sh
```

Para esta release, substitua `vX.Y.Z` por `v0.1.0-beta.2`.

Enquanto `https://get.contabase.net` não estiver ativo, use `https://get-contabase.pages.dev` como endpoint público do canal.

### Source/build local

```bash
git clone --branch v0.1.0-beta.2 https://github.com/contabase-app/contabase.git
cd contabase
chmod +x scripts/install-contabase-source.sh
sudo ./scripts/install-contabase-source.sh
```
