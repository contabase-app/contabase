# ContaBase v0.1.0-beta.3 - Beta publica controlada

> **Status:** Em preparacao; ainda nao publicada
>
> **Estagio:** Beta publica controlada, nao stable

## Principais mudancas desde v0.1.0-beta.2

### Adicionado
- **Manifests de canal:** `stable` e `beta` passam a ser descritos por manifests no canal publico de instalacao.
- **Estado unificado de instalacao:** instalacoes passam a registrar metodo, canal e versao instalada em `/etc/contabase/install-state.env`.
- **`cb-update` por canal:** update sem argumento respeita o canal instalado; `--channel beta|stable` troca canal explicitamente; versao manual vira modo pinned.
- **Modo explicito de acesso:** `CONTABASE_ACCESS_MODE=local|lan|proxy` passa a registrar o contrato de exposicao da instancia, sem tratar `ALLOWED_HOSTS` como liberacao suficiente para HTTP remoto.

### Alterado
- **`/install.sh` representa stable:** enquanto nao houver stable publicada, o canal stable falha de forma clara e orienta usar beta ou `CONTABASE_VERSION`.
- **`/beta/install.sh` resolve manifest beta:** a beta atual passa a ser obtida de `public/channels/beta.json`, sem versao hardcoded no bootstrap.
- **Gate de release reforcado:** `publish-public.sh check` valida os manifests locais do `contabase-canal` antes de aprovar o snapshot.

### Corrigido
- **Migracao de beta existente:** instalacoes legadas em tag beta, incluindo `v0.1.0-beta.2`, sao classificadas como canal `beta` ao migrar para o novo estado.
- **Acesso LAN por IP privado:** instalacoes guiadas sem proxy com `APP_BASE_URL=http://IP_PRIVADO:PORTA` agora persistem `CONTABASE_ACCESS_MODE=lan` e permitem HTTP apenas para esse IP privado RFC1918; IP publico e dominio HTTP sem proxy continuam bloqueados.
- **Versionamento espalhado:** scripts ativos deixam de usar exemplos hardcoded de betas antigas como fonte operacional.

## Contrato de acesso

| Modo | Quando usar | Regra |
|---|---|---|
| `local` | Teste na propria maquina | HTTP somente em `localhost`/`127.0.0.1` |
| `lan` | Rede privada sem proxy | HTTP somente para o IP RFC1918 configurado em `APP_BASE_URL` e em `ALLOWED_HOSTS` |
| `proxy` | Dominio com HTTPS | Acesso remoto via HTTPS reconhecido por proxy confiavel em `TRUSTED_PROXIES` |

`ALLOWED_HOSTS` continua obrigatorio como allowlist de Host, mas nao autoriza sozinho HTTP remoto.

## Contrato de instalacao

### Canal stable

```bash
curl -fsSLo /tmp/contabase-install.sh https://get-contabase.pages.dev/install.sh
bash /tmp/contabase-install.sh
```

Enquanto nao houver stable publicada, este fluxo deve abortar com mensagem clara.

### Canal beta

```bash
curl -fsSLo /tmp/contabase-install.sh https://get-contabase.pages.dev/beta/install.sh
bash /tmp/contabase-install.sh
```

Este fluxo instala a versao apontada por `public/channels/beta.json`.

### Versao especifica

```bash
curl -fsSLo /tmp/contabase-install.sh https://get-contabase.pages.dev/install.sh
CONTABASE_VERSION=v0.1.0-beta.3 bash /tmp/contabase-install.sh
```

Instalacoes com `CONTABASE_VERSION` explicito entram em modo `pinned`.

## Atualizacao

```bash
sudo cb-update
sudo cb-update --channel beta
sudo cb-update --channel stable
sudo cb-update v0.1.0-beta.3
```

- `stable` atualiza para a ultima stable publicada.
- `beta` atualiza para a ultima beta publicada.
- `pinned` exige uma versao explicita ou troca explicita de canal.
- beta nunca migra para stable automaticamente.
